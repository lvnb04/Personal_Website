# NeuralNet Home — Design Spec

_Date: 2026-06-18 · Phase 6 · Status: approved, ready for implementation plan_

This is the design for the home page neural-network nav (`CLAUDE.md §10`, §13).
It records the five decisions confirmed during brainstorming and the build plan.
Where this conflicts with anything, the order of authority is:
`decisions.md` > this spec > `CLAUDE.md`. This spec does not contradict either —
it refines them.

## Confirmed decisions (this session)

| # | Decision | Choice |
|---|---|---|
| 1 | Code organization | **Self-contained `NeuralNet.astro`** — SVG build + animation JS + styles in one file; only `seededRandom.ts` extracted. Independent of `AmbientNet` (different renderer). |
| 2 | Click-pass speed | **Scenic ~0.8s**, exposed as a tunable constant (`CLICK_PASS_MS`). Per `decisions.md §5`. |

## 1. Architecture & files

| File | Purpose |
|---|---|
| `src/lib/seededRandom.ts` | Deterministic PRNG (Mulberry32). Fixed seed → identical network every visit (the author's "logo"). Exports `seededRandom(seed: number): () => number` returning a float in `[0,1)`. Pure, no deps. |
| `src/components/NeuralNet.astro` | **Self-contained** (`CLAUDE.md §12`). SVG generated by a build function from `site.sections`; all animation JS + scoped `<style>` live here. Imports only `site` (nodes/labels) and `seededRandom` (edge topology). |
| `src/pages/index.astro` | Replaces the temp placeholder. Full-bleed net + overlaid chrome. **No Header** (the net is the nav). |

**Renderer:** SVG, not canvas — deliberately different from `AmbientNet` (canvas force-graph).
SVG is required for the spec's affordances: real `<a href>` nodes, `stroke-dashoffset`
edge-draw, keyboard focus, JS-off fallback. No shared abstraction between the two
components; they share nothing but "dots and lines."

**Tunables → `constants.ts`** (per `decisions.md §7`, no inline magic numbers).
A `NEURAL_NET` config block, e.g.:

```ts
export const NEURAL_NET = {
  seed: 20260618,            // fixed → same network every visit
  layersDesktop: [4, 7, 7, 7, 5], // inputs derived from sections; kept explicit for clarity
  layersMobile:  [4, 5, 5, 5],
  edgeDensity: 0.65,         // fraction of possible edges that exist
  edgeOpacity: [0.25, 0.6],  // resting opacity range (seeded per edge)
  clickPassMs: 800,          // scenic; hard ceiling 1200
  idleIntervalMs: [3000, 4000],
  misclassRate: 0.12,        // ~10–15%, hover only
  mobileBreakpoint: 767,
};
```

`layers[0]` is asserted against `site.sections.length` at build/runtime; outputs =
`inputs + 1`. Adding a 5th section in `site.ts` grows the net automatically.

## 2. Network construction (seeded SVG)

`buildNetwork(layers, seed)` runs in the client `<script>` on load, and again when the
desktop/mobile breakpoint is crossed. It produces a pure data model, then renders SVG.

**Data model (computed, never hand-authored):**
- `layers = [inputs, 7, 7, 7, outputs]`; `inputs = site.sections.length` (4),
  `outputs = inputs + 1` (5).
- **Nodes:** evenly distributed along the cross-axis per layer, centered. Input nodes
  carry `{label, href}` from `site.sections`; output nodes = same labels **+ `/dev/null`**
  (the one joke). Hidden nodes anonymous. Hidden-layer radius dips mid-net (7→5→7 px) for depth.
- **Edges:** for each adjacent layer pair, the seeded PRNG decides each possible edge's
  existence (~65% density) and resting opacity (0.25–0.6). Same seed → same topology forever.

**Rendering:** one `<svg>` with a fixed `viewBox` (scales to any container). Edges drawn
first (behind), then nodes. Input + output nodes are real `<a href>` wrapping the circle
**and** its label text (whole thing is the click target). Hidden nodes are plain `<circle>`.

**Brightness hierarchy** (dimmest → brightest, on near-black `--bg`):
`--edge` (1px, varied opacity) < hidden nodes (`--surface` fill, `--text-faint` border)
< input labels (`--text-dim`) < **active = `--accent` iris, used nowhere at rest.**

**JS-off / a11y baseline:** nodes are real `<a>`, so the net is navigable with zero JS
(renders static, links work). `aria-label` on the svg describes it. Nodes are
keyboard-focusable with a visible focus ring.

## 3. Interaction state machine (desktop)

States: `IDLE → HOVERING → SETTLED → COMMITTING`. A single active-animation token guards
against overlaps — a new interaction cancels the prior. Built in layered order:

**(1) Static** — seeded net renders, links work. *Shippable alone.*

**(2) Click input = commit + navigate** *(critical path — built second)*
- Plain left-click only → `preventDefault`, fast clean forward pass (`clickPassMs ≈ 800`),
  **always correct** (no misclassification on click), winner locks `--accent`, then
  `location.assign(href)` ~250ms after settle.
- **Escape hatches:** 2nd click OR any key during pass → navigate now.
  Middle/Ctrl/Cmd-click → do NOT preventDefault (browser opens new tab; no animation).

**(3) Idle** — every 3–4s a faint pulse (~30% brightness) travels one random valid path;
outputs rest near-uniform (`0.33/0.33/0.33/0.01`). Never misclassifies. Pauses on
tab-hidden (`visibilitychange`, like `AmbientNet`).

**(4) Hover = theater, NO navigation** (hover ≠ intent)
- L→R cascade ~150ms/layer (~800ms total). Edges "draw" via `stroke-dashoffset`
  (`dasharray` = path length, offset length→0). Nodes flash as they fire.
- Light only ~10–15 edges (2–3 branching paths to the winner, incl. one main). Never all.
- Softmax readout jitters through 3–4 values then settles; winner → `--accent` (~0.93).
- After a correct settle: winning **output** becomes a live link (border glows, gains
  quiet `open ↵`). Lingers ~2–3s after cursor leaves, then fades to idle.
- `Enter` after a settled pass accepts the prediction → navigate.

**(5) Backprop misclassification — built LAST** (~12% of hovers, HOVER ONLY)
- Lands on a WRONG output in `--amber`, LOW confidence (~0.55–0.70, correct class as
  runner-up — a wrong model is *unsure*, never confidently wrong). Caption
  `loss: 0.87 — backpropagating…`.
- Signal runs BACKWARD (reversed draw, amber); 2–3 edge opacities shift. Forward pass
  auto-re-runs faster, lands CORRECT (green); caption `loss: 0.04 ✓`.
- Guards: **never on first hover; never twice in a row; never confidently wrong.**

**Reduced motion:** `prefers-reduced-motion: reduce` → no animation; hover statically
highlights the path; click navigates instantly.

**Three doors** (only the first two are hinted; the rest are discovered):
click input (fast) · hover then click the lit output (scenic) · `Enter` after settle.

**Hint line:** `hover to think · click to go` (mono, `--text-ghost`).

## 4. Mobile (`CLAUDE.md §13`)

**Detection — each method for its one job:**
- **JS rebuild trigger:** `matchMedia("(max-width: 767px)")` + a `change` listener →
  `rebuildNetwork()` on the breakpoint cross (tablet rotation safe).
- **Hover vs tap:** `matchMedia("(hover: hover)")` decides whether hover-theater is wired
  at all. Never `ontouchstart`; never UA sniffing.

**Mobile network = a different build, NOT a rotated SVG:**
- `layers = [4, 5, 5, 5]`, `direction = "vertical"`, top-to-bottom flow.
- **Inputs:** horizontal row at the top, label BELOW each node, `publications → papers`
  for width. Input-label font shrinks at this breakpoint. Verify all four clear at 375px.
- **Output:** NO full softmax row — a single result-readout pill at the bottom
  (`→ projects · 0.93`).
- **Interaction:** tap = quick pass → navigate. Second tap = skip. NO hover-theater, NO
  backprop on mobile. Idle pulse still runs, traveling downward.
- **Tap targets ≥44px** (circle + label + padding).

**Universal hard requirements verified for this page (`§13`):** viewport meta present;
zero horizontal scroll at 375px (`body.scrollWidth === window.innerWidth`); nav never
wraps. The home page has no Header/Tiles/TOC, so most of §13's grid/TOC items don't apply —
the home-specific checks are: four input labels fit, readout pill visible, no collision.
